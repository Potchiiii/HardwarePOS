✓ DATABASE TABLES CREATED
=========================

TABLE: purchase_orders
├─ id (PRIMARY KEY)
├─ order_number (UNIQUE)
├─ item_id (FOREIGN KEY → inventory)
├─ item_name
├─ brand
├─ quantity
├─ status (ENUM: pending, approved, ordered, received)
├─ created_by (FOREIGN KEY → users)
├─ created_at
├─ updated_at
└─ notes

TABLE: notifications
├─ id (PRIMARY KEY)
├─ order_number
├─ item_name
├─ quantity
├─ message
├─ created_by (FOREIGN KEY → users)
├─ created_at
└─ sent_to_staff (TINYINT: 0=pending, 1=sent)

✓ WORKFLOW EXAMPLE
==================
1. Procurement user opens dashboard
   → Sees 3 low stock items, 2 pending orders

2. Clicks "View Low Stock Items"
   → Sees item "Drill 710 watts" with 43 units (below threshold of 50)

3. Clicks "Order" button for that item
   → Modal opens, user enters quantity (50)
   → Order created as ORD-2025-001

4. Goes to "Purchase Orders"
   → Updates status from Pending → Approved → Ordered

5. When stock arrives, goes to "Notify Staff"
   → Creates notification
   → Fills in order number, item name, quantity, message
   → Sends notification

6. Inventory staff sees notification in their dashboard
